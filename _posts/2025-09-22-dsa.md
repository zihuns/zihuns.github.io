# 🏆 사내 AI 해커톤: 매장관리 & 온보딩 Slack Bot

---

## 1. 팀 소개

- 팀명: 디셀프 (Diself) / DSA: **'Display**'와 **'Self', 'AI'**의 합성어로, 매장 전시(Display) 관련 문의나 문제를 담당자 스스로(Self)가 해결할 수 있도록 지원하는 AI 문의 대응봇을 만들고자 만든 팀입니다.
- 챗봇 이름: @dsa - **d**isplay + **s**elf + **a**i (키보드로 소환하기 쉽게 우겨넣었습니다)
- 매장 전시 관련 문의 처리 효율화 와 자가 해결 능력 향상을 목표로 합니다.

## 2. 배경

- **반복적인 문의 발생:** 회사 내 **#자유소통, #coop-전시-기획-개발소통** 등 다양한 채널에서 유사한 매장 전시 관련 문의가 지속적으로 발생하고 있습니다.
- **단순 문의 다수:** 대부분의 문의는 단순 데이터 조회나 설정 확인만으로 해결 가능한 사안으로 파악되었습니다.
- **주요 원인은 기본적 오류:** 매장 문제는 주로 설정 오류, 데이터 누락 등 기본적인 운영 과정에서의 실수로 인해 발생하고 있습니다.
- **부서 간 협업 부담:** 문제 해결을 위해 프론트엔드(FE), 백엔드(BE), 운영팀 간 소통이 필요하여 해결 시간 지연 및 비용 증가로 이어지고 있습니다.
- **개발 효율성 저하:** 모든 문의가 개발자나 PO에게 전달되어 개발 생산성이 저하되고 운영 대응에 과도한 리소스가 소모되고 있습니다.

## 3. 개선 목표 및 기대 효과

### 핵심 추진 목표

- 
    - **신속한 문제 식별 및 자동화:** 매장 전시 운영 과정에서 발생하는 반복적 문의와 결함을 신속하게 식별하고, AI 기술을 활용하여 원인 분석 및 대응 프로세스를 자동화하겠습니다.
    - **효율적 커뮤니케이션:** 부서 간 커뮤니케이션 비용을 최소화하고, 문제 발생 시 담당 영역(FE/BE/운영)별로 빠르게 분류 및 전달하는 체계를 구축하겠습니다.
    - **지식 관리 체계화:** 문의 이력과 해결 사례를 체계적으로 축적하여, 신규 이슈 발생 시 즉각적인 참고 및 대응이 가능하도록 지원하겠습니다.
    - **품질 및 경험 개선:** 운영 효율성 증대와 더불어, 매장 전시 품질 및 고객 경험을 지속적으로 개선할 수 있는 기반을 마련하겠습니다.

### 기대 효과

- 
    - **매장 운영 안정성 향상:** 모듈 노출 기준 및 운영 결함 자동 진단 시스템 도입으로 문제 발생률을 대폭 감소시킬 것으로 기대됩니다.
    - **업무 효율성 증대:** 문의 접수부터 원인 파악, 담당자 연결까지 전 과정의 자동화를 통해 업무 처리 속도와 대응 정확도를 크게 개선할 수 있을 것입니다.
    - **데이터 기반 신속 대응:** 반복적 문의와 결함에 대한 체계적인 데이터 축적 및 분석으로 신규 이슈 발생 시 대응 시간을 현저히 단축할 수 있을 것입니다.
    - **조직 리소스 최적화:** 부서 간 소통 비용 절감 및 운영 리소스의 효율적 배분을 통해 인력 운영 효율성을 개선할 수 있을 것으로 기대됩니다.
    - **고객 경험 제고:** 매장 전시 운영 품질 향상을 통해 최종 사용자 만족도를 크게 향상시킬 수 있을 것으로 예상됩니다.

## 4. 프로젝트 진행 계획

### 1. 현황 분석 및 문제 유형 분류

- 
    - **데이터 수집 및 분석:** 매장 전시 운영 현황, 주요 결함 및 문의 유형 데이터 수집
    - **분류 체계 구축:** FE/BE/운영별 원인 분류 기준 정립 및 소통 프로세스 설계
    - **핵심 지표 설정:** 문의 해결 시간, 반복 문의율, 담당자 연결 정확도 등 성과 측정 지표 선정

### 2. AI 챗봇 요구사항 정의 및 설계

- 
    - **분류 로직 설계:** 반복적 문의/결함 자동 분류 로직 설계
    - **통합 인터페이스 설계:** Slack/Jira/Confluence 연동 구조 설계 및 API 명세 작성
    - **자연어 처리 모델 선정:** BERT/GPT 계열 모델을 활용한 의도 분류 및 개체명 인식 시스템 설계

### 3. 프로토타입 개발 및 기준 적용

- 
    - **챗봇 기능 개발:** 매장 모듈 노출 기준(최소/최대) 적용 챗봇 기능 개발
    - **자동화 기능 구현:** 문의 의도 및 주요 정보(키워드, 항목 등) 인식, 자동 원인 분석 및 담당자 연결 기능 구현
    - **데이터베이스 구축:** MongoDB를 활용한 문의 및 해결책 데이터베이스 설계 및 구축

### 4. 단계별 확대 및 실증 테스트

- 
    - **기능 확장:** 카테고리, 필터, 랭킹 등 다양한 노출 기준 적용 확대
    - **시나리오 테스트:** 실제 운영 데이터 기반 시나리오 테스트 및 성능 검증
    - **개선 사이클:** 피드백 수집 및 기능 개선 반복
    - **벡터 검색 고도화:** Elasticsearch와 FAISS를 활용한 유사 문의 검색 기능 구현

### 5. 최종 배포 및 운영 체계 구축

- 
    - **통합 운영:** Slack, Jira, Confluence 등 협업 도구와 통합 운영
    - **AI 모델 고도화:** 지속적 데이터 축적 및 AI 모델 고도화
    - **모니터링 체계:** 매장 전시 운영 품질 및 고객 경험 개선을 위한 모니터링 체계 구축

## 6. 아키텍처 및 기술 스택

- **프론트엔드:** Vue2, TypeScript 기반의 SPA로 매장 전시 문의 UI 및 챗봇 인터페이스 구현
- **백엔드:** Spring Framework, Java 기반 REST API 서버로 비즈니스 로직 처리
- **AI 엔진:** Spring AI와 LangChain을 활용한 자연어 처리 및 의도 분석 파이프라인 구축
- **검색 엔진:** Elasticsearch와 FAISS를 활용한 벡터 기반 유사 문의 검색 및 빠른 데이터 조회
- **데이터베이스:** MongoDB(비정형 데이터)와 RDB(MySQL/PostgreSQL)를 활용한 하이브리드 데이터 저장 구조
- **메시징/협업:** Slack Bot API를 통한 실시간 문의 접수 및 자동 응답 처리
- **인프라:** Docker와 Kubernetes를 활용한 마이크로서비스 아키텍처 구현 및 확장성 확보
- **DevOps:** Jira, Confluence API 연동을 통한 이슈 추적 및 지식베이스 자동 업데이트

7. 매장 전시 문의 자동 대응 시스템 Flow Chart

| **프로세스**                                           | **사용 기술**                    |
| ------------------------------------------------------ | -------------------------------- |
| 사용자(운영/담당자)가 Slack 채널에 매장 전시 문의 등록 |
| ↓                                                      | Slack API                        |
| Slack Bot이 문의 메시지 수신                           |
| ↓                                                      | Slack Bot API                    |
| Spring 백엔드 서버로 문의 데이터 전달                  |
| ↓                                                      | Spring Framework, Java           |
| AI 분석 모듈에서 문의 의도 및 주요 정보 추출           |
| ↓                                                      | Spring AI, LangChain, TensorFlow |
| 문의 유형 분류 및 원인(운영/FE/BE) 자동 분석           |
| ↓                                                      | Spring AI, 자연어 처리           |
| 문의 처리:                                             |
- 자주 묻는 질문/단순 결함: DB/설정값 조회 후 자동 응답
- 복잡/신규 이슈: 담당자 자동 태깅 및 요약 전달
↓ | MongoDB/RDB, Elasticsearch, FAISS |
| 응답 결과를 Slack 채널에 자동 전송 | Slack API |
| 문의/응답/해결 이력 DB에 저장 및 지속적 학습 데이터로 활용 | MongoDB, MySQL/PostgreSQL |
